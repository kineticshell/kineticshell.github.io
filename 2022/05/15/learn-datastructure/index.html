数据结构学习笔记（更新中）

C语言 数据结构 CS
干货
发布日期:   2022-05-15

第一章：绪论

定义

相互之间存在一种或多种特定关系的数据元素的集合

计算机解决具体问题的步骤

  • 首先从具体问题抽象出一个数学模型
  • 设计一个解此问题的算法
  • 编程序、进行测试，调整直至得到最终解答

数学模型的分类

  • 数值计算类
  • 非数据计算类
    • 树结构
    • 线性结构
    • 图结构

名词解释

数据

  • 所有能输入到计算机中并能被计算机程序加工、处理的符号的总称

  • 如整数、实数、字符、声音、图像、图形等

数据元素

  • 数据的基本单位

  • 在计算机中作为一个整体进行考虑和处理

数据项

  • 数据不可分割的最小单位

  • 一个数据元素可以由多个数据项组成

  • 如姓名、年龄等

数据对象

  • 由性质相同（类型相同）的数据元素组成的集合

  • 数据对象是数据的一个子集

  • 由4个整数组成的数据对象

    • D1={20, -30, 88， 45}

数据结构

  • 相互之间存在一种或多种特定关系的数据元素的集合

  • 数据元素之间的关系称为结构

  • 四类基本结构

    • 集合
    • 线性结构
    • 树形结构
    • 图状结构

  • 按逻辑结构分类

    • 线性结构
      • 线性表
      • 栈
      • 队列
      • 数组
      • 广义表
      • 字符串
    • 非线性结构
      • 树和二叉树
      • 图

数据的存储结构

  • 数据在计算机存储器中的映像

  • 也被称为存储表示、物理结构、物理表示

  • 分类

    • 顺序存储结构（向量、一维数组）
    • 非顺序存储结构（单链表：分配不一定连续的空间）

数据类型

  • 一个值的集合和定义在这个值上的一组操作的总称
    • 原子类型（int, char, float等）
    • 结构类型（线性表、数组、树等）

抽象数据类型

  • 是指一个数学模型以及定义在该模型上的一组操作，它是对数据结构逻辑上的定义，与计算机的实现无关

  • 抽象数据类型可以用一个三元组表示：（D，S，P）

    • D表示数据对象
    • S是D中数据元素之间的关系集
    • P是对D中数据元素的基本操作

算法的基本特征

定义

算法是对特定问题求解步骤的一中描述，还不是程序

五个特征

  • 有穷性：在有限步骤/时间后终止
  • 确定性：每条指令/步骤都没有二义性，具有明确含义
  • 可行性：算法中的操作都是已经实现的基本运算执行有限次来实现的
  • 输入：有0或多个输入量
  • 输出：有至少一个输出量

算法设计要求

  • 正确性

    • 无语法错误
    • 对n组输入产生正确结果
    • 对特殊输入产生正确结果
    • 对所有输入产生正确结果

  • 可读性

    • 算法主要是为了人的阅读与交流

  • 健壮性

    • 不同的输入都要有相应的反应

  • 高效与低存储量

    • 时间复杂度
    • 空间复杂度

算法的描述工具

  • 自然语言
  • 程序设计语言
  • 流程图
  • 伪代码
  • 类C语言

算法的主体

  • 函数
    • 用于表示算法

算法分析的相关标准

时间复杂度

定义

算法/程序中基本操作/语句重复执行的次数的总和

求解

设n为求解问题的规模，基本操作/语句的执行次数总和称为语句频度，记作**f(n)**。

执行下面的步骤：

  • 去掉f(n)中所有加法常数
  • 只保留最高阶项

**时间复杂度记作T(n)**，T(n)=O(f(n))。

第二章：线性表

定义

  • 由n(n>=0)个数据元素组成的有限序列。

对于L=(a1, a2, …, ai-1, a1, ai+1, …, an)

  • ai-1在ai之前，是ai的直接前驱
  • ai+1在ai之后，是ai的直接后继
  • a1没有前驱
  • an没有后继
  • ai(1<i<n)有且仅有一个直接前驱和直接后驱

线性表的基本操作

  • 删除
  • 插入
  • 排序
  • 查找
  • 查找并删除
  • 合并
  • 复制

线性表的存储结构

顺序分配

在内存中是紧挨着存储的，使用的是连续的存储空间。

在定义时需定义长度或元素位置，取其一即可。

元素所占的空间和表长可以合并为C语言中的一个结构类型。

超出定义好的分配空间会导致溢出。

动态分配

当超出容量时，分配更大的空间存储。引入指针的概念来实现此效果。

在顺序表中插入元素

将插入元素后的每个元素后移一位，新元素填补空位。

插入一个元素时移动元素的平均值是n/2

在顺序表中删除元素

将删除元素后的每个元素前移一位。

顺序存储结构

优点

  • 是一种随机存储结构，存取任何一个元素的时间是一个常数，速度快。

  • 结构简单，逻辑上相邻的元素也是物理上相邻的。

  • 不需要使用指针，节省存储空间

缺点

  • 插入和删除元素需要大量时间

  • 需要一块连续的存储空间

  • 插入元素时可能发生溢出

链式存储结构

单链表

定义

线性表中的每个结点分散地储存在内存中，先后依次用指针串联起来

分为不带表头结点和带表头结点两种类型

  • 不带表头结点
    • HEAD（头指针）指向首节点
      • 结点由DATA（数据域）和NEXT（指针域/链域）组成
    • 当HEAD==NULL时味为空表
    • 当为非空表时HEAD指向非空数据
  • 带表头结点
    • HEAD指向表头结点
      • 表头结点的DATA为空，NEXT域指向首节点
    • HEAD->NEXT域为空则是空表

单链表的结点结构

与前者相同，但每一次接收新输入时改用HEAD指向新元素。

循环链表

一般形式
  • 带表头结点的非空单循环单链表
    • HEAD -> next != HEAD且H != NULL
  • 带表头结点的空单循环单链表
    • HEAD -> next == HEAD且H != NULL
只设尾指针的循环链表

  • 非空表

    • tail指向表尾结点
    • tail -> data == an
    • tail -> next指向表头结点
    • tail -> next -> next指向首结点
    • tail -> next -> next -> data == a1

  • 空表

    • tail -> next == tail

便于将两个循环链表首尾相连

双向链表

结构

PRIOR DATA NEXT
前驱 数据 后继

非空表

HEAD -> next -> data == a0
HEAD -> prior -> data == an
HEAD -> next -> prior == HEAD -> prior -> next == NULL

空表

HEAD -> next == HEAD -> prior == NULL

双向循环链表

  • 空表
    • HEAD -> next == HEAD -> prior == HEAD
  • 非空表
    • 设p指向a1，则p -> next指向a2，p -> next -> prior指向a1
    • p -> next -> prior == p -> prior -> next == p

第三章：栈与队列

栈

一种限定在表尾进行插入删除操作的线性表

名词说明

  • 进栈
    • 插入一个元素到栈中
  • 出栈
    • 从栈删除一个元素
  • 栈顶
    • 允许插入、删除元素的一端（即表尾）
  • 栈顶元素
    • 处于栈顶的元素
  • 栈底
    • 栈中不允许插入/删除元素的一端
  • 空栈
    • 不含元素的栈
  • 后进先出原则

顺序栈

顺序栈的描述

栈元素与顶指针合并为同一个结构

  • 约定

    • 栈元素空间[0…maxleng]
    • top指向元素的上一个空位置
    • top是栈顶标志，根据约定由top找栈顶元素

  • 存储分配方式

    • 静态分配
    • 动态分配

链式栈

将链表的指针颠倒过来，即令top指向an

链式栈的进栈

p = (struct node *)malloc(sizeof(struct node));//分配新结点
p -> data = e;//数据域赋值
p -> next = top;//next域指向头部
top = p;//头指针指向p

链式栈的退栈

p = top;//p指向栈顶首结点
top = top -> next;//栈顶指针指向栈顶下一个元素
free(p);//释放原栈顶占用的内存空间

栈的应用

  • 数制转换
  • 判断表达式中{和}是否匹配
  • 表达式求值

队列

  • 只允许在一端插入元素，另一端删除元素的线性表
  • 队首head/front
  • 队尾rear/tail
  • 入队/出队=新元素插到队尾/删除队首元素
  • 又名先进先出表

队列的头和尾

  • Q.front队头首指针，指向队头结点
  • Q.rear队尾指针，指向队尾结点
  • Q.front -> data不放元素
  • 空队列时首尾指针都指向首节点，首节点next域为空

生成空队列

#define LENG sizeof(Qnode)
LinkQueue InitLQ(){
  LinkQueue Q;
  Q.front=Q.rear=(QueuePtr)malloc(LENG);
  Q.front -> next = NULL;
  return Q;
}

插入新元素

Qnode p = (Qnode *)malloc(LENG);//建立新结点
p -> data = e;//给新结点的next域赋值
p -> next = NULL;//新结点的next域为空
Q.rear -> next = p;//尾元素next域指向新结点
Q.rear = p;//尾指针指向新结点

删除元素

/*非空队列*/
Qnode *p = Q.front;//指向待删除首节点
(*e) = p -> data;//取出data
Q.front ->next = p -> next;//首指针指向第二个结点
free(p);//释放目标结点的内存

/*单元素队列*/
Qnode *p = Q.front;
(*e) = p -> data;
Q.front ->next = Q.rear;//首指针指向尾指针
free(p);
